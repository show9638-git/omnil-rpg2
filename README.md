# 全零〈オムニル〉— 継承の境界・装備庫拡張版

白・黒・虹を宿す3人で周回・育成・職業解放・装備厳選を行う、GitHub Pages対応の静的WebハクスラRPGです。

## 起動

### GitHub Pages

このフォルダの中身をリポジトリ直下へアップロードし、GitHub **Settings → Pages** で `main` ブランチの `/(root)` を公開対象にします。公開URLを開けば遊べます。

### ローカル確認

```bash
python -m http.server 8080
```

ブラウザで `http://localhost:8080` を開いてください。

## 主な追加内容

- 装備所持上限：**10,000件**
- 6レアリティ：コモン／アンコモン／レア／エピック／レジェンダリー／ユニーク
- 装備庫の検索・カテゴリ・レアリティ・タグ・レベル・並び順フィルター
- 50件単位のページ送りで、10,000件でも一覧操作を軽く維持
- 表示中選択、絞り込み全選択、選択反転、選択解除
- 一括売却・分解・保護・保護解除・お気に入り・ジャンク指定
- 装備中／保護中／お気に入り／ユニークを安全に除外する一括処理
- 周回のレアリティ別自動売却・自動分解・自動保持
- カテゴリ、タグ、効果、装備レベルによる周回中の保持条件
- 虹全・白零・黒零それぞれの変更不可・枠消費なしベーススキル
- 主人公画像を `assets/characters/<hero>/` から差し替え可能なマニフェスト構造

## 主人公ベーススキル

- **虹全：三彩共鳴** — 白系と黒系を繋いで共鳴を蓄積し、虹追撃・全体防壁を発動。
- **白零：創生の護り** — 回復・防壁・浄化から味方の被ダメージ軽減と防壁破壊時の再生を作る。
- **黒零：終焉刻印** — 状態異常・弱体中の敵へ追撃と低HP処刑を行う。

ベーススキルは外せず、技能枠・コスト枠を使いません。

## 周回の自動整理

**周回 → 自動整理設定** で、レアリティごとに `保持 / 自動売却 / 自動分解` を選べます。

保持条件に指定したカテゴリ・タグ・効果・装備レベルは、レアリティ設定より優先されます。ユニークは常に保持です。

## アセット差し替え

主人公素材は以下のファイルを置き換えてください。ファイル名と透過PNG形式を維持すれば、ゲームコードの変更は不要です。

```text
assets/characters/kozen/idle.png
assets/characters/kozen/attack.png
assets/characters/kozen/hurt.png
assets/characters/kozen/down.png
assets/characters/kozen/victory.png

assets/characters/hakurei/...
assets/characters/kokurei/...
```

- 推奨：透過PNG、同じ縦横比、ドット絵はニアレストネイバーで拡大
- 表示倍率・パス定義：`assets/manifest.js`
- 敵・地域・装備アイコンも同マニフェスト経由で参照
- 画像欠損時はゲーム全体を止めず、ブラウザのコンソールに警告を出します

## セーブ

進行はブラウザの `localStorage` に自動保存されます。設定画面からJSONとして書き出し・読み込み可能です。破損データや不正な装備IDは安全に除外して読み込みます。
